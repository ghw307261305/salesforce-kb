# Agentforce 讲师版课件大纲（12章）

- 版本: v1
- 最后更新: 2026-02-28
- 适用对象: 企业内训讲师 / 技术布道 / 项目 onboarding
- 课件粒度: 每章 30-45 页

## Summary

本大纲用于快速制作“可授课、可演示、可考核”的 Agentforce 全套课件。  
每章按统一版式组织：`导入 -> 核心知识 -> 实操演示 -> 练习 -> 小结`。

## Decision / Changes

1. 采用 12 章对应 12 套主课件的结构，和教材章节一一映射。
2. 每章提供“建议页数 + 时间预算 + 互动方式 + Demo 建议”。
3. 引入“讲师备注区”，用于记录常见问答与课堂风险提示。

## 1. 课件总体设计规范

### 1.1 单章结构（建议）

| 模块 | 建议页数 |
| --- | --- |
| 封面与目标 | 2-3 |
| 关键概念 | 8-12 |
| 方法与流程 | 8-10 |
| Demo/案例 | 6-10 |
| 练习与答疑 | 4-6 |
| 小结与作业 | 2-4 |

### 1.2 视觉与讲授规范

1. 每页只讲一个结论点，避免信息堆叠。
2. 图优先于字，流程图优先于长段落。
3. 关键术语采用“中文 + 英文关键词”。
4. 每章至少 2 页“常见错误与纠偏”。

## 3. 讲师讲稿与备课建议

1. 每章准备 1 页“讲师开场脚本”。
2. 每章准备 1 页“高频问答与标准答复”。
3. 每章准备 1 个 Demo 录屏备用方案（防止现场网络问题）。
4. 每章准备 1 页“课堂时间控制点”。

## Risks / Known Issues

1. 每章页数较多，若无时间控制容易超时。
2. 部分章节依赖环境演示，网络/权限异常会影响授课节奏。
3. 若讲师背景差异大，内容深浅不一会影响学习体验。
