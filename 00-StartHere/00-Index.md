# 00-StartHere インデックス

ここはナレッジベースの開始地点です。まず運用ルールを固め、`01-Clouds` と `02-Shared` に継続的に知見を蓄積します。

## 最初にやること

1. [01-Study-Roadmap.md](./01-Study-Roadmap.md) を開き、今週の学習目標と成果物を決める。
2. [03-My-Tooling.md](./03-My-Tooling.md) でローカル環境と定番コマンドを整理する。
3. [04-My-Standards.md](./04-My-Standards.md) で命名・記述ルールを統一する。
4. 毎日のメモは [99-Inbox/_to-triage.md](../99-Inbox/_to-triage.md) に入れ、週末に分類する。

## ナビゲーション

- クラウド別資産: [01-Clouds](../01-Clouds)
- 共通再利用資産: [02-Shared](../02-Shared)
- ロール別プレイブック: [03-Roles](../03-Roles)
- プロジェクト成果物: [04-Projects](../04-Projects)
- 参照スニペット: [05-Reference](../05-Reference)

## コンテンツの流れ

- プロジェクト固有の事象は `04-Projects/<Project>/...` に記録する。
- 再利用可能な内容は `01-Clouds` または `02-Shared` に昇格する。
- 同じ内容は複製せず、リンク参照でつなぐ。

## 週次ルーチン

- 平日: `99-Inbox` に素早く収集。
- 週末: Inbox を整理し、Cloud/Shared に振り分け。
- 週次レビュー: 学習計画と優先順位を更新。
