# 04 記述標準

## 命名規則

- ファイル名は意味が分かる短い名前を使用。
- 主要ディレクトリは `00-Index` / `01-Concepts` / `02-HowTo` などの固定構造を維持。
- ADR は `ADR-XXX` の連番で管理。

## 記述規則

- 1ファイル1テーマ。
- 冒頭に「対象・前提・結論」を明記。
- 手順書は「前提条件 / 手順 / 検証」を必須化。
- 同じ説明を複数箇所に書かず、リンクで参照。

## 品質規則

- 結論は検証可能であること（ログ・コマンド・画面・参照リンク）。
- Troubleshooting は「症状 / 原因 / 対処 / 再発防止」を含める。
- Pattern は「適用条件 / 利点 / トレードオフ」を含める。

## 配置規則

- `01-Clouds`: クラウド固有の知識。
- `02-Shared`: 跨クラウドで再利用する知識。
- `03-Roles`: 進め方、テンプレート、チェックリスト。
- `04-Projects`: 案件固有の判断、履歴、成果。

## レビュー規則

- 週次: リンク切れ、重複、命名の見直し。
- 月次: 高頻度参照記事を QuickLinks に追加。
